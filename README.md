# D-ETCIF

## 1. 项目介绍

### 项目简介

> D-ETCIF is Data-driven_Experimental_Teaching_with_Continuous_Intelligent_Feedback system
> 即数据驱动的实验教学伴随式评价与智能反馈系统

围绕数据驱动的实验教学伴随式评价与智能反馈系统展开研究，聚焦“Python数据可视化”课程当前实验教学中评价滞后、反馈针对性不足等实际问题，遵循五条核心链路，涵盖离线初始化、全流程数据采集处理、双层知识图谱构建更新、智能三级反馈生成及双端应用交互，其中业务逻辑层包含数据处理、知识建模、智能分析反馈三大引擎，应用层面向师生提供差异化功能，最终通过全流程闭环设计，实现精准学情诊断、个性化反馈与高效教学辅助，助力提升实验教学质量与效率。

### 技术栈

- 后端技术栈：go + gin + gorm + WebSocket
- 前端技术栈：pnpm + Vite + React + TypeScript + Recharts + Tailwind CSS
- Python分析服务：FastAPI + Node2Vec + LightFM + 资源TopK推荐
- 数据层：Neo4j图数据库 + Mysql关系型数据库

## 2. 系统基础链路设计

### 2.1 离线初始化链路（链路1）

Python：从教材/指导书/错误手册抽取三元组 → 写入 领域图谱（Neo4j）
Go：GORM AutoMigrate → 初始化 MySQL 表；导入资源库与班级/用户基础数据
Python：预热 Node2Vec / LightFM / 规则引擎（或规则表缓存）

### 2.2 实验全流程采集处理（链路2）

前端：埋点事件通过 WS/HTTP 发送
Go：事件接收 → 结构化 → 清洗 → 分流
MySQL：操作日志、测验、评分等强事务
Neo4j：知识点映射、置信度更新（可同步写 / 异步队列写）

### 2.3 双层图谱构建与更新（链路3）

领域图谱：静态，只在初始化/版本迭代更新
学生认知图谱：实验过程中实时更新（事件驱动）
向量化：
领域图谱 Node2Vec：离线或定时
学生薄弱点向量：实时/批处理

### 2.4 智能分析与三级反馈（链路4）

实时反馈：Go 调用规则匹配（Python 或规则表）→ WS 推送弹窗
路径建议：Go 请求 Python 推荐服务（LightFM）→ 返回路径步骤+代码片段
总结诊断：实验结束触发 TopK → 学习报告/资源推荐

### 2.5 师生端应用展示（链路5）

学生端：学习空间/个人图谱/学习报告
教师端：班级仪表盘/学生行为分析/预警/热力图
